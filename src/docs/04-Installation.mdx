# Installation

This package is intended for use within the [Comet Components plugin](https://packagist.org/packages/doubleedesign/comet-canvas-blocks) for the WordPress block editor, so generally you'll want to just go ahead and install that.

---

### Plugin implementation overview

Below is an overview of how this package is implemented in that plugin, as a high-level guide for how you might implement it in other plugins or themes, or to help with troubleshooting the Comet plugin on a site.

Add to the dependencies in `package.json`:

```json
"dependencies": {
  "@doubleedesign/comet-gutenberg-controls": "github:doubleedesign/comet-gutenberg-controls"
}
```

#### JavaScript

Create a wrapping editor component like this:

```jsx
/* global wp */
import { addFilter } from '@wordpress/hooks';
import { CometBlockControls } from '@doubleedesign/comet-gutenberg-controls';

wp.domReady(() => {
	addFilter(
		'editor.BlockEdit',
		'comet-plugin-blocks/custom-controls',
		(BlockEdit) => (props) => {
			return (
				<div className="comet-block-edit-wrapper" data-block={props.name}>
					<CometBlockControls BlockEdit={BlockEdit} {...props} />
				</div>
			);
		});
});
```

Compile it using [Rollup](https://rollupjs.org/). Please see the current [build config](https://github.com/doubleedesign/comet-components/blob/master/packages/comet-plugin-blocks/rollup.config.js) in the Comet plugin for an example config file.

**Important:** You also need to recompile this script after updating the version of the `@doubleedesign/comet-gutenberg-controls` package in your plugin/theme.

Add a build script to `package.json` like this:

```json
"scripts": {
  "build": "rollup -c"
}
```

And running it like so:

```powershell
npm run build
```

Load the compiled JavaScript into the block editor via the `enqueue_block_editor_assets` PHP hook:

```php
add_action('enqueue_block_editor_assets', function() {
    wp_enqueue_script(
        'comet-blocks-custom-controls',
        "$pluginDir/src/editor/CustomControls.dist.js",
        ['wp-blocks', 'wp-element', 'wp-editor', 'comet-block-registry'],
        '1.0.0',
        true
    );
});
```

#### CSS

Load the CSS into the block editor by importing it into an SCSS file that is compiled and loaded into the block editor, so the code all gets brought into the compiled CSS (and it doesn't try to import from the `node_modules` directory in production).

```scss
@use '../../../node_modules/@doubleedesign/comet-gutenberg-controls/dist/style.css';

```

**Important:** You also need to recompile this stylesheet after updating the version of the `@doubleedesign/comet-gutenberg-controls` package in your plugin/theme.

#### PHP

Use `wp_localize_script` to define the client-side global `comet` object and its values so the editor components know what options and values to display for things like colours, and default values on a per-block/component level. Comet Components does this through a global configuration object and a range of filter hooks, but it would be feasible to process `theme.json` values into the correct format if preferred.

You can find the definition of the object in [types.ts](./src/types.ts), and an example of `wp_localize_script` in the [BlockEditorConfig.php](https://github.com/doubleedesign/comet-components/blob/master/packages/comet-plugin-blocks/src/BlockEditorConfig.php) file in the Comet plugin.
